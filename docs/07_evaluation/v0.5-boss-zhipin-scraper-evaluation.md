## 1. 结论

该项目工程完整度较好，能够在隔离环境安装、运行 CLI 和通过上游测试；真实 CDP
链路也能连接本机 Chrome，并能把 BOSS `code 37` 识别为限制状态后立即停止。

但它不适合原样加入 `campus-job-agent` 作为 `source.discover_jobs` 的 Tool 或运行时
backend，原因是：

1. 搜索 API 原始响应没有先进入本项目 Evidence Store；页面 JavaScript 先将响应投影为
   精简职位字段，然后 Python 才保存结果，不满足 raw-before-parse。
2. 运行方式依赖独立 Chrome profile 和 CDP 端口，不能消费现有
   `local-secret://boss_jobs/default` credential ref。
3. 没有本项目的 `SourceBatch`、`SourceDocument`、`SourceRunReceipt` 和稳定错误 envelope；
   CLI 主要通过终端文本与退出码交互。
4. 包含覆盖 `document.hidden` / `visibilityState`、模拟滚动和鼠标移动等降低自动化
   可见性的行为，超出本项目“不规避风控、不模拟用户行为”的安全边界。
5. 默认详情采集和单次最多 500 次 API 请求的范围明显大于 v0.5 live smoke 的最小、
   用户主动、低频只读边界。

可参考的部分仅限于确定性代码思路：BOSS 限制状态分类、风控即停、隔离 profile 生命周期、
城市码表、字段映射和 JD 登录墙检测。参考不意味着复制运行路径或放宽安全边界。

## 3. 安全与证据审计

### 3.1 做得较好的部分

- MIT 许可证明确，提交较新，上游含 76 个无需联网的测试。
- 默认创建持久隔离 Chrome profile，不复制主 Chrome 登录数据。
- 已知 `31/37`、HTTP `403/429` 和限制关键词能够被分类为 restricted。
- 登录探测遇到 confirmed restriction 会立即停止。
- 页数上限、请求总预算、翻页等待和默认禁用 DOM 薪资 fallback 均有明确实现。
- 详情页能区分登录墙、导航壳与可用 JD，避免把截断内容标成完整详情。

### 3.2 阻断准入的问题

#### Raw evidence 丢失

列表请求的 `xhr.responseText` 在浏览器页面内直接解析，随后只返回映射后的职位数组。
原始 HTTP 状态、响应头和完整响应 body 没有先归档。后续写出的 JSON 已是解析产物，无法
满足 Evidence Store 的不可变 raw artifact、hash、parser version 和 replay 要求。

#### 自动化规避行为

上游对后台页面覆盖 visibility 属性，并实现随机滚动、随机停顿和鼠标抖动。即使目的只是
改善页面渲染，这些行为也会改变站点对自动化环境的可见信号，不能进入本项目默认 transport。

#### Tool 契约不兼容

上游函数持有 Chrome 进程、CDP target、文件输出和终端日志的生命周期；错误多以自然语言
打印和进程退出表达。直接用 subprocess 包装会丢失 `authentication_required`、
`rate_limited`、`source_changed`、`policy_blocked` 等本项目状态及 raw artifact refs。

#### 凭证模型不兼容

上游依赖 Chrome profile 内的 Cookie，可选复制主 Chrome Cookie 数据库；本项目 Tool 输入
只能接收 credential ref，并由本地 credential service 在 adapter 边界解析。两者不能直接
共用，且不应为了接入而把 profile 路径或 Cookie 暴露给 Graph/LLM。

## 4. 若未来重新评估，必须先满足

1. 新建最小 transport，而不是 subprocess 包装完整 CLI；删除 visibility override、模拟滚动、
   鼠标移动和任何 stealth 语义。
2. 每次 HTTP 响应先以完整 bytes 和必要元数据归档，再调用确定性 parser。
3. 只接受 `credential_ref`，秘密值仅在本地 adapter 内解析，不复制主 Chrome profile。
4. 返回本项目 `SourceBatch`/`ToolResult`，明确映射 auth、restricted/rate limit、empty、
   source changed 和 network timeout。
5. 默认一页、无详情、低请求率；收到 `31/37/403/429` 后立即结束整个 SourceRun。
6. 增加 fixture replay、无凭据、凭据过期、风控即停、脱敏和 raw-before-parse 测试。
7. 在正常人工登录且平台未返回限制状态时，再执行一次不超过 5 条结果的 opt-in live smoke。

满足这些条件后，适合重新评估的是“受本项目契约约束的新 adapter”，而不是当前上游 CLI
整体进入运行时依赖。
