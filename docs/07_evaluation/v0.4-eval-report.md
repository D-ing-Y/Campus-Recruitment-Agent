## 故障结果

- 扫描件/无文本层 PDF：原文件已归档，解析返回 `unsupported_input`，Graph 请求支持格式或允许 skip。
- 非法 LLM JSON/schema：有限重试；失败记录进入 LLM trace，充分性评价使用 deterministic fallback。
- Profile Repository/Tool 写入失败：不得创建画像，run 标为 failed。
- Checkpoint 写入失败：抛出 `checkpoint_error`，不声称任务可恢复；已完成的 Evidence 写入仍保持可审计和幂等。
- 错误 request/user/thread/action：Evidence Store 零写入并拒绝恢复。
- 同 response ID 的不同 payload：返回 `idempotency_conflict`。

## 限制

- PDF 仅支持有文本层的常规文档；不包含 OCR、表格还原或复杂多栏版面分析。
- 默认 deterministic/mock provider 用于离线可重复验收；本报告没有真实远程 LLM 的延迟、token 成本或开放集质量数据。
- SQLite Evidence Store 与 SQLite checkpointer 面向本地单机流程，不代表多 worker 或分布式事务能力。
- v0.4 以 Python library/test runtime 暴露 subgraph，尚未新增专用 v0.4 CLI 或 Web UI。
- 不包含岗位画像、岗位检索、匹配、RAG、分布式存储或 Multi-Agent。
