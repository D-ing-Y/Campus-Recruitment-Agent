# Project Charter

## 项目名称

Campus Job Agent：证据驱动的双画像求职 Agent。

## 项目目标

构建一个基于 LangGraph 的垂直求职 Agent。系统以原始材料为事实来源，持续构建和校准：

- 候选人画像：用户当前能力、教育、科研、项目、实习和可证明证据；
- 求职意图：岗位、城市、薪资、行业、硬性约束和可协商条件；
- 岗位需求画像：具体岗位与岗位族的资格、能力、加分项和招聘筛选信号。

系统比较候选人画像和岗位需求画像，识别能力差距、证据差距、偏好冲突和认知不确定性，并在用户确认下完成岗位探索、准备计划和笔面试反馈更新。

## 项目价值

产品价值：

- 帮助目标不清晰的学生用真实市场证据形成和验证职业选择；
- 把简历、项目、论文、JD、面经和反馈连接成可追溯的长期决策闭环；
- 避免无证据画像和一次性学习建议造成级联错误。

工程与面试价值：

- 从零实现 LangGraph 状态图、subgraph、conditional edge、interrupt、checkpoint 和 durable execution；
- 实现严格结构化输出、工具契约、Evidence Store、Hybrid RAG、长期记忆和评估；
- 实现本地存储到 PostgreSQL、对象存储、向量存储和异步队列的演进；
- 在确有收益时实现主 Agent 与 Sub-Agent 的动态委派；
- 能够用基线、指标和失败案例解释每项技术选择，而不是只展示技术名词。

## 核心原则

- 证据优先：原始材料是事实源，画像、差距和计划都是可重建的派生状态。
- 结论可追溯：事实性结论必须关联 evidence 和 provenance；推断必须显式标记。
- 确定性骨架、动态决策：代码控制契约、安全和评分，LLM决定如何消除当前不确定性。
- 人类掌握职业决策：Agent提出假设与行动，用户确认目标、偏好和高影响判断。
- 文档驱动：桌面端 Codex 维护设计文档，VSCode 端 Codex依据文档实现代码。
- 渐进复杂化：前沿技术必须解决真实问题，并与简单基线比较。
- 可观测和可恢复：所有长任务可追踪、可中断、可恢复、可评估。
- 合规采集：不绕过验证码或反爬；保留来源、时间、置信度和原始归档。

## 技术方向

| 能力 | 项目角色 |
| --- | --- |
| LangGraph | 父图、子图、循环、条件路由、checkpoint、interrupt 和恢复 |
| LLM Provider | 可替换模型调用、结构化输出、重试、缓存和调用追踪 |
| Evidence Store | 原始文件、片段、claim、来源和画像版本的事实基础 |
| Hybrid RAG | 在大规模材料中检索与当前未知项最相关的证据 |
| 分布式存储 | 支持多 worker、长任务、大文件、向量索引和故障恢复 |
| Multi-Agent | 仅用于动态并行、上下文隔离和长任务分解 |
| Eval / Observability | 验证 groundedness、检索、图路由、成本、延迟和恢复能力 |

## 已完成基础

- v0.1 已验证 LangGraph Mini Runtime、ToolRegistry、trace、report 和测试骨架。
- v0.2 已验证 LLM Provider、结构化输出、Pydantic 校验、有限重试、缓存和 LLM trace。
- 两个版本均作为后续架构的底座保留；业务状态和图拓扑从 v0.3 起升级。

## 非目标

- 不训练基础大模型。
- 不把招聘网站自动投递作为当前核心目标。
- 不把单一匹配百分比描述为 Offer 概率。
- 不通过一次面试反馈直接改写整个岗位族画像。
- 不为了展示而无条件引入 RAG、分布式系统或 Multi-Agent。
- 不复刻完整 Codex 或 DeerFlow 平台。

## 成功标准

- 用户可从材料上传走到岗位选择、准备计划和反馈更新。
- 画像、差距和计划的关键结论可回溯到真实证据。
- 用户调整偏好后系统能回退、重检索并保持历史状态。
- 长任务支持中断、恢复、预算和失败定位。
- RAG、分布式存储和 Multi-Agent 均有可运行实现、明确业务用途和量化评估。
