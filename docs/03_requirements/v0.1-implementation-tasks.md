# v0.1 Mini Agent Runtime 实现任务清单

状态：Draft
日期：2026-07-08
关联需求：`docs/03_requirements/v0.1-mini-runtime.md`
关联 RFC：`docs/04_rfc/0001-mini-agent-runtime.md`
关联 ADR：`docs/05_adr/0001-use-langgraph-for-runtime.md`

## 1. 实现原则

- 只实现 v0.1 Mini Agent Runtime。
- 不接入真实招聘网站。
- 不接入真实 LLM API。
- 不实现 RAG、Memory、多 Agent、Web UI、服务器部署。
- 不引入真实 cookie、登录态、验证码处理或反爬逻辑。
- 使用 LangGraph 最小能力：`StateGraph`、`START`、`END`、`add_node`、`add_edge`、`compile`、`invoke`。
- 工具调用必须经过 `ToolRegistry`，节点不得直接调用具体工具函数。
- 所有输出文件写入 `data/runs/<run_id>/` 和 `data/reports/`。

## 2. 推荐实现顺序

```text
项目配置
  -> schema
  -> state 初始化
  -> tool contract / registry / mock tool
  -> planner
  -> executor
  -> verifier
  -> report writer
  -> LangGraph workflow
  -> CLI
  -> unit tests
  -> integration tests
  -> eval tests
  -> README 更新
```

### Task 03：实现 AgentState 与初始化

- 定义 LangGraph 使用的状态结构。
- 提供初始状态构造函数。

涉及文件：

```text
src/campus_job_agent/agent/state.py
```

要求：

- 定义 `AgentState`。
- 实现 `create_initial_state(user_input: str) -> AgentState`。
- 生成 `run_id`。
- 初始化 `plan`、`tool_results`、`trace`、`errors`。
- 设置 `output_dir = data/runs/<run_id>`。

验收：

- 初始 state 包含 RFC 要求字段。
- `user_input` 保留原始文本。

### Task 04：实现 Tool 协议与 ToolRegistry

目标：

- 建立工具注册和调用机制。

涉及文件：

```text
src/campus_job_agent/tools/base.py
src/campus_job_agent/tools/registry.py
```

要求：

- 定义 `Tool` protocol 或 base class。
- 实现 `ToolRegistry.register`。
- 实现 `ToolRegistry.get`。
- 实现 `ToolRegistry.run`。
- 未注册工具必须返回 failed `ToolResult`，不要抛出未处理异常。

验收：

- 可以注册并调用工具。
- 未注册工具返回结构化失败结果。

### Task 05：实现 mock_job_search

目标：

- 提供 v0.1 用于验证工具调用链路的 mock tool。

涉及文件：

```text
src/campus_job_agent/tools/mock.py
```

要求：

- 工具名固定为 `mock_job_search`。
- 输入使用 `role_query`、`city`、`graduation_year`。
- 输出至少 2 条 records。
- 第一条记录匹配成都 AI Agent。
- 第二条记录允许包含噪声。
- `evidence_ids` 字段存在，可为空。

验收：

- 工具返回符合 `ToolResult`。
- records 非空。

### Task 06：实现 planner

目标：

- 实现目标解析和计划生成。

涉及文件：

```text
src/campus_job_agent/agent/planner.py
```

要求：

- 实现 `parse_goal_text(user_input: str) -> ParsedGoal`。
- 实现 `create_plan(parsed_goal: ParsedGoal) -> list[PlanTask]`。
- v0.1 使用规则解析。
- 默认计划调用 `mock_job_search`。

解析规则：

- 包含 `成都` -> `city = "成都"`，否则 `unknown`。
- 包含 `AI Agent` -> `role_query = "AI Agent"`。
- 包含 `智能体` 且不包含 `AI Agent` -> `role_query = "智能体"`。
- 包含 `2027` -> `graduation_year = "2027"`，否则 `unknown`。

验收：

- 输入 `"成都 AI Agent 2027 秋招"` 能解析出三个核心字段。
- 未知字段填入 `unknown`。
- 生成计划包含 `mock_job_search`。

### Task 07：实现 executor

目标：

- 执行 plan 中的工具任务。

涉及文件：

```text
src/campus_job_agent/agent/executor.py
```

要求：

- 实现 `execute_plan(plan: list[PlanTask], registry: ToolRegistry) -> list[ToolResult]`。
- 每个 task 通过 `registry.run(task.tool_name, task.args)` 执行。
- 单个工具失败不终止整个 plan。

验收：

- 至少能执行 `mock_job_search`。
- 未注册工具返回 failed 结果。

### Task 08：实现 verifier

目标：

- 校验运行结果是否满足 v0.1 最低要求。

涉及文件：

```text
src/campus_job_agent/agent/verifier.py
```

要求：

- 实现 `verify_state(state: AgentState) -> VerificationResult`。
- 检查项：
  - `role_query_present`
  - `plan_non_empty`
  - `tool_results_non_empty`
  - `tool_result_fields_valid`

验收：

- 成功路径 `passed = true`。
- 缺少 plan 或 tool_results 时 `passed = false`。

### Task 09：实现 report writer

验收：

## 4. 建议验收命令

在 `campus-job-agent/` 目录运行：

```bash
python -m venv .venv
source .venv/bin/activate
pip install -e ".[dev]"
python apps/cli/main.py run "成都 AI Agent 2027 秋招"
pytest
```

若暂未配置 editable install，可临时使用：

```bash
PYTHONPATH=src python apps/cli/main.py run "成都 AI Agent 2027 秋招"
PYTHONPATH=src pytest
```
