# v0.3 实现任务清单

状态：Proposed
日期：2026-07-15
关联需求：`docs/03_requirements/v0.3-evidence-domain-foundation.md`
关联 RFC：`docs/04_rfc/0003-evidence-domain-foundation.md`

## 1. 文档与契约

- [ ] 确认 evidence、candidate、intent、role、gap schema。
- [ ] 确认 SQLite 表、迁移和文件目录。
- [ ] 确认 Claim 类型、引用校验和幂等键。
- [ ] 准备匿名简历、项目说明、JD 和错误 Claim fixture。

## 2. Schema

- [ ] 新增 `schemas/evidence.py`。
- [ ] 新增 `schemas/candidate.py`。
- [ ] 新增 `schemas/intent.py`。
- [ ] 新增 `schemas/role.py`。
- [ ] 新增 `schemas/gap.py`。
- [ ] 更新 schema 导出和版本字段。

## 3. Storage 与 Repository

- [ ] 定义 `BlobStore`、`EvidenceRepository`、`ProfileRepository` protocol。
- [ ] 实现本地 blob store 与原子写入。
- [ ] 实现 SQLite 连接和 schema migration。
- [ ] 实现 artifact/fragment/claim/profile repository。
- [ ] 实现 hash 去重、幂等写入和失败清理。

## 4. Evidence Pipeline

- [ ] 实现 Artifact ingestion。
- [ ] 实现 TXT/Markdown/HTML fixture 文本读取；PDF 只完成登记或简单提取。
- [ ] 实现确定性 Fragmenter 和 locator。
- [ ] 实现 ClaimValidator。
- [ ] 实现 ProfileSnapshot 的最小保存与读取。

## 5. LLM 复用升级

- [ ] 从 `parse_search_goal_with_llm` 提炼泛型 structured output helper。
- [ ] 保持 v0.2 SearchGoal 路径兼容。
- [ ] 新增版本化 Claim extractor prompt/schema。
- [ ] 复用 mock provider 测试合法、非法、重试和 provider error。
- [ ] 确保 cache key 包含 prompt/schema/fragment hash。

## 6. Capability Ontology

- [ ] 定义首版 ontology 文件和版本。
- [ ] 实现 canonical name/alias 查找。
- [ ] 未匹配项保留 raw label，不静默映射。

## 7. Trace、Report 与 Eval

- [ ] trace 增加 artifact/fragment/claim 摘要，不记录完整敏感文本。
- [ ] 输出 evidence pipeline report。
- [ ] 实现 evidence trace、unsupported claim、locator、dedup 指标。
- [ ] 生成 v0.3 eval report。

## 8. 测试与验收

- [ ] schema 单元测试。
- [ ] blob/repository/migration 单元测试。
- [ ] dedup、幂等和失败清理测试。
- [ ] Claim 引用存在性与类型约束测试。
- [ ] Artifact→Fragment→Claim 集成测试。
- [ ] Candidate/Intent/Role snapshot round-trip 测试。
- [ ] v0.1/v0.2 全量回归测试。
- [ ] 更新 README、roadmap、contracts 和验收报告。

## 9. 明确禁止

- 不提交真实简历、Cookie、API key 或个人敏感数据。
- 不在 Agent 节点直接写 SQL 或本地路径。
- 不让 LLM 绕过 ClaimValidator 直接写入确认画像。
- 不在 v0.3 同时实现 RAG、分布式存储、真实爬虫或 Multi-Agent。
- 不删除旧 Mini Runtime 和 SearchGoal smoke test。
