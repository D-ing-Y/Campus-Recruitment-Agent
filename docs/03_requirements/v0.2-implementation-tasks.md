## 1. 实现原则

- 只实现 v0.2 LLM Provider 与结构化输出能力。
- 不改变 v0.1 LangGraph 主流程拓扑。
- v0.2 只把 LLM 接入 `parse_goal` 节点，其他节点暂不改为 LLM 驱动。
- 默认 provider 必须是 `mock`，测试不得依赖真实 API key 和网络。
- OpenAI-compatible provider 只实现最小 Chat Completions HTTP 调用。
- Provider 层只负责模型请求和原始响应，不理解业务 schema。
- JSON 解析、Pydantic 校验、重试、缓存必须放在 structured output 层。
- 不允许 API key、Authorization header、完整环境变量进入 state、trace、cache、report 或测试快照。
- 不接入真实招聘网站。
- 不实现 RAG、Memory、多 Agent、Web UI、服务器部署。
- 不引入 LangChain、OpenAI Agents SDK 或具体厂商 SDK。

## 3. 每个文件实现什么

### 3.1 `src/campus_job_agent/llm/__init__.py`

实现：

- 导出 v0.2 LLM 层常用对象。
- 至少导出 `LLMProvider`、`LLMConfig`、`MockLLMProvider`、`OpenAICompatibleProvider`。

不允许：

- 不写业务逻辑。
- 不读取环境变量。
- 不执行 provider 初始化副作用。

### 3.2 `src/campus_job_agent/llm/base.py`

实现：

- `LLMProvider` protocol。
- `LLMRequest`、`LLMResponse` 可放在此文件，或从 `schemas/llm.py` 导入后复用。
- provider 层统一异常基类可放在此文件。

不允许：

- 不依赖 `AgentState`。
- 不解析 `SearchGoal`。
- 不写 cache。
- 不写 report。

### 3.3 `src/campus_job_agent/llm/config.py`

实现：

- `load_llm_config()`。
- 从环境变量读取：
  - `CAMPUS_AGENT_LLM_PROVIDER`
  - `OPENAI_API_KEY`
  - `OPENAI_BASE_URL`
  - `OPENAI_MODEL`
  - `CAMPUS_AGENT_LLM_CACHE_ENABLED`
  - `CAMPUS_AGENT_LLM_FALLBACK_TO_RULE_PARSER`
- 默认 provider 为 `mock`。
- `openai_compatible` 缺少必要配置时返回结构化配置错误。

不允许：

- 不打印 API key。
- 不强制要求存在 `.env` 文件。
- 不在 import 时读取环境变量。

### 3.4 `src/campus_job_agent/llm/mock.py`

实现：

- `MockLLMProvider`。
- 支持模式：
  - `valid_json`
  - `invalid_json_then_valid`
  - `schema_error_then_valid`
  - `always_invalid_json`
  - `provider_error`
- 记录调用次数，便于测试 cache hit 时 provider 未被调用。

不允许：

- 不访问网络。
- 不读取 API key。
- 不使用随机输出。

### 3.5 `src/campus_job_agent/llm/openai_compatible.py`

实现：

- `OpenAICompatibleProvider`。
- 使用 `httpx` 调用 `<base_url>/chat/completions`。
- 支持 `model`、`messages`、`temperature`、`response_format`、`timeout`。
- 从响应中抽取 assistant message text、usage、非敏感 metadata。
- 将 HTTP、超时、鉴权失败等错误映射为 provider error。

不允许：

- 不实现 streaming。
- 不实现 function calling / tool calling。
- 不引入 OpenAI 官方 SDK 或其他厂商 SDK。
- 不把 Authorization header 写入任何输出对象。
- 不在 provider 层做业务 schema 校验。

### 3.6 `src/campus_job_agent/llm/cache.py`

实现：

- `LLMCache`。
- 按 hash 分文件读写 JSON cache。
- `make_cache_key(...)`。
- cache key 包含 provider、model、prompt name、prompt version、schema version、messages。
- cache value 保存 raw output、parsed JSON、created_at、usage 等非敏感信息。
- cache 损坏时视为 miss，并返回可记录的 cache error 摘要。

不允许：

- API key 不进入 cache key。
- API key 不进入 cache value。
- 不使用全局内存 cache 代替文件 cache。
- 不因为 cache 写入失败导致主流程失败。

### 3.7 `src/campus_job_agent/llm/structured.py`

实现：

- `parse_search_goal_with_llm(...)`。
- 结构化输出主流程：
  - 构造 prompt messages。
  - 计算 cache key。
  - cache hit 时读取 raw output。
  - cache miss 时调用 provider。
  - `json.loads`。
  - `SearchGoal.model_validate`。
  - JSON 或 schema 失败时最多重试 1 次。
  - 返回 `SearchGoal` 和 `LLMCallRecord` 列表。

不允许：

- 不直接访问 HTTP。
- 不直接读取 API key。
- 不写 Markdown report。
- 不吞掉重试后仍失败的错误。
- 不在未显式配置时 fallback 到规则解析。

### 3.8 `src/campus_job_agent/prompts/__init__.py`

实现：

- prompt package 初始化。
- 可导出 `goal_parser` 相关函数。

不允许：

- 不放具体 provider 配置。
- 不读取环境变量。

### 3.9 `src/campus_job_agent/prompts/goal_parser.py`

实现：

- `PROMPT_NAME = "goal_parser"`。
- `PROMPT_VERSION = "v1"`。
- `SCHEMA_VERSION = "v0.2"`。
- `build_goal_parser_messages(user_input: str)`。
- `build_goal_parser_retry_messages(user_input, previous_output, error_summary)`。
- prompt 明确要求只输出 JSON。
- prompt 明确未知字段填 `"unknown"`、`[]` 或 `null`。

不允许：

- 不要求模型编造用户没有提供的信息。
- 不要求模型输出 Markdown。
- 不把 API key 或 provider 信息写入 prompt。

### 3.10 `src/campus_job_agent/schemas/goal.py`

实现：

- 新增 `SearchGoal`。
- 保留现有 `ParsedGoal` 和 `PlanTask`。

  - `industries`

  - `confidence`
  - `warnings`

不允许：

## 7. 验收命令

在 `campus-job-agent/` 目录运行。

安装依赖：

```bash
python -m venv .venv
source .venv/bin/activate
pip install -e ".[dev]"
```

默认 mock provider 运行：

```bash
python apps/cli/main.py run "成都 AI Agent 2027 秋招"
```

完整测试：

```bash
pytest
```

定向测试：

```bash
pytest tests/unit/test_search_goal_schema.py
pytest tests/unit/test_llm_config.py
pytest tests/unit/test_llm_cache.py
pytest tests/unit/test_llm_mock_provider.py
pytest tests/unit/test_structured_output.py
pytest tests/integration/test_v02_llm_goal_parsing.py
pytest tests/evals/test_v02_llm_eval.py
```

如果暂未使用 editable install，可临时运行：

```bash
PYTHONPATH=src python apps/cli/main.py run "成都 AI Agent 2027 秋招"
PYTHONPATH=src pytest
```

真实 OpenAI-compatible provider smoke test 只作为手动验证，不作为自动测试必需项：
