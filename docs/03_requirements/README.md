# Version Requirements

本目录按版本维护需求文档。

命名示例：

- `v0.1-mini-runtime.md`
- `v0.2-llm-provider.md`
- `v0.3-evidence-domain-foundation.md`
- `v0.4-candidate-profile-graph.md`
- `v0.5-role-profile-graph.md`

每份需求文档必须说明：

- 背景
- 目标
- 非目标
- 输入输出
- 模块范围
- 验收标准
- 测试与 eval 要求

已完成版本的需求文档作为历史记录保留；新架构不得通过回写旧文档伪造连续性，应新增版本文档和 ADR。
